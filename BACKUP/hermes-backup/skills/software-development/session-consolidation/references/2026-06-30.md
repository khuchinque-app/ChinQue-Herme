# Session: 2026-06-30 — Tittil Design + Auto-Chat App

## What was done

### 1. Tittil Website Design (Green Theme)
- Extracted menu data from `tittil.cloud/tittil/` (53 items across 5 categories: Pempek, Mains, Ricebowl, Sides, Drinks)
- Built full HTML/CSS/JS single-page food ordering site
- Initial design used warm/orange palette (inferred from food context)
- **User corrected to green theme** — rebuilt all CSS variables (`--brown-*` → `--green-*`), hero gradient, nav background, accent colors
- Final palette: `#F4FAF3` cream, `#4CAF50` accent, forest green text
- Key features: sticky category filter, dual pricing (USD/KHR), cart panel, Telegram order, floating cart bar, confirmation modal

### 2. Two-Person Auto-Chat Application
- **Architecture diagram** — dark-themed SVG showing: User A/B → Internet → Nginx/Caddy → FastAPI (WebSocket) → SQLite + Ollama AI
- **Backend** — FastAPI + WebSocket + SQLite + JWT auth + auto-reply engine
- **Frontend** — Single HTML file (dark theme), real-time chat, auto-reply settings modal, rule-based keyword replies
- **Docker** — Dockerfile + docker-compose.yml
- **Dependency fix** — bcrypt 5.0.0 broke passlib 1.7.4; replaced password hashing with native hashlib+salt approach
- **Verification** — app runs on port 8000, health check passes, login works

### Key Lessons / Pitfalls

| Lesson | Type |
|--------|------|
| Never assume brand theme from context (food=warm). Ask or extract actual colors. | Design workflow |
| bcrypt 5.0 breaks passlib — use hashlib+salt instead for self-contained apps | Dependency fix |
| WebSocket chat needs 3 message patterns: send-to-receiver, echo-to-sender, auto-reply | App architecture |
| Docker compose with profiles lets Ollama be optional (`--profile ai`) | Deployment pattern |
