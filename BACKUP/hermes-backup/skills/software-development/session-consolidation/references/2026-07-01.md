## File Manager UI Pattern (captured because `popular-web-designs` is bundled)

**Correction:** Initial cloud storage UI used a card grid (`.file-grid` with
`repeat(auto-fill, minmax(200px, 1fr))`). User rejected it — wanted table rows.

**Lesson:** File managers / cloud storage UIs must use a **table layout**
(sortable columns: icon+name, size, date, actions), not cards. Layout
elements: left sidebar (Files, Notes, Settings), breadcrumb navigation,
upload button in top bar, per-file download/delete on hover.

**When cards ARE correct:** Media galleries, portfolios, product grids —
anything where the visual preview IS the primary content.

## Flask+Gunicorn+Docker Web App Pattern

Full-stack web app built from scratch:
- Single Docker container with Python 3.11-slim + Gunicorn
- Flask with blueprints (auth, files, notes)
- SQLite database for users, files, notes
- File upload with extension whitelist + blacklist
- 5GB max upload (MAX_CONTENT_LENGTH + Gunicorn --timeout 600)
- UFW locking to port 22 + 8443 only
- Volume mount for live CSS/JS reload in dev
