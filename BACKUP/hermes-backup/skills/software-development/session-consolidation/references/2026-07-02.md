## Database Schema (MenuItem)
```sql
CREATE TABLE MenuItem (
    id                TEXT PRIMARY KEY,  -- Format: c<hex> (e.g. c5f5e0ed1f9b8387deb68b374)
    name              TEXT NOT NULL,
    description       TEXT,
    price             TEXT NOT NULL,     -- Stored as TEXT (decimal string like "3.50")
    image             TEXT,              -- Path like "assets/images/pempek/pempek_01.jpg"
    isAvailable       INTEGER DEFAULT 1,
    categoryId        TEXT NOT NULL,     -- FK to Category.id
    restaurantId      TEXT NOT NULL,     -- FK to Restaurant.id
    stockQuantity     INTEGER,
    ...
);
```

### Menu Rendering Flow
1. `index.php` renders HTML shell (header, hero, toolbar, menu-grid container, cart drawer, chat widget)
2. `app.js` calls `loadMenu()` on DOMContentLoaded:
   - Fetches categories: `GET /api/menu/categories.php?restaurant=tittil`
   - Fetches items: `GET /api/menu/items.php?restaurant=tittil&available=1&limit=100`
   - Fetches restaurant info: `GET /api/menu/restaurant_info.php?restaurant=tittil`
3. `renderItems()` builds cards dynamically — images loaded from `i.image` path (line 218)
4. Images with `onerror` fallback to emoji placeholder (line 218 onerror handler)

### Key Finding: Image rendering
In `app.js` line 218:
```javascript
${i.image ? `<img src="${esc(i.image)}" ... onerror="...">` : '<div class="img-placeholder">🍽</div>'}
```
Images are loaded from the `image` path stored in the DB. If the image fails to load, it falls back to emoji. The image paths in DB are relative to the storefront root (e.g. `assets/images/pempek/pempek_01.jpg`).

### Image Mapping Strategy
- Multiple items share the same image by category (practical for 53 items)
- Pempek: 3 images distributed across 22 items
- Mains: 2 images + 1 soup image across 11 items
- Drinks: 1 image across 11 items
- Sides: 1 image across 4 items
- Ricebowl: 1 image across 4 items

### Download Pattern
```bash
curl -sL "https://images.pexels.com/photos/<ID>/pexels-photo-<ID>.jpeg?cs=srgb&dl=<filename>&fm=jpg" -o pempek_01.jpg
```

### DB Update Pattern
```sql
UPDATE MenuItem SET image = 'assets/images/pempek/pempek_01.jpg' WHERE id = '<item-id>';
```
